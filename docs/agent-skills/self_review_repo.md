# 自审仓库

只有在空闲自驱、用户要求、或创作者相关场景中需要看自己代码时使用。目标是发现具体可改进点，不是为了展示内部实现。

流程:

1. 明确这次自审的范围和一个可验证问题。
2. 用 `workspace_bash cwd=repo` 只读查看代码和文档。
3. 先读 `docs/README.md` 找入口，再按问题读具体文件。
4. 只用只读命令，不改源码、不写项目文件。
5. 如果发现明确问题，私聊创作者：问题是什么、依据在哪、建议怎么改。

边界:

- 不要把内部路径、配置、白名单等细节发到群里。
- 不要在没有依据时说“代码有 bug”。
- 不要把一次自审拉成长篇报告；给 1-3 个具体点就够。
- 如果需要真正改代码，让创作者明确授权。
