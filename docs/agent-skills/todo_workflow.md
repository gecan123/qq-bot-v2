# Todo 工作流

已经决定执行且预计至少三个相互依赖步骤、跨 capability，或需要调查→执行→验证闭环时，用 `todo action=update` 写一个短计划。计划只服务当前进程内的短期工作，不是长期记忆；本 skill 只解释 todo 的更新规则，不替代 todo 本身，也不负责其他专项规则。

规则:

- 每条 todo 只写一个动作。
- 同一时间最多一个 `in_progress`。
- 开始做哪一步，就把那一步设为 `in_progress`。
- 完成一步后，立刻 `todo action=update` 标成 `completed`，再把下一步设为 `in_progress`。
- 如果任务变了，提交完整的新列表，不要只补一条增量。

适合:

- 读多个文件后再判断。
- 需要先查工具、再执行、再验证。
- 用户要求“先分析/对照/优化”这种容易跑偏的任务。

不适合:

- 单句聊天回复。
- 长期目标、跨重启项目管理。
- 要记住某个人或某个群的信息；那应该用 `memory`。
