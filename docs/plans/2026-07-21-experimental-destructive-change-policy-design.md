# 实验性项目破坏性改动默认策略

## 背景

仓库已声明这是实验性新项目，并倾向于干净的目标模型，但尚未明确破坏性改动是否需要备份，以及规则是否适用于外部或生产数据。

## 决策

- 对仓库源码、schema 和本地开发数据，除非用户明确要求，否则允许直接做破坏性改动，无需备份、兼容层或历史迁移。
- 继续优先选择干净的目标模型，不围绕旧 bridge 或 adapter 设计。
- 该默认值不授权删除生产数据或外部服务数据。

## 落点与验证

合并强化 `AGENTS.md` 的现有“实验性新项目”条款，并将相同内容同步到 `CLAUDE.md`。修改后检查两文件字节级一致，并运行 `pnpm repo-check`。
