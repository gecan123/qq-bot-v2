# Luna 自主兴趣与生活记录设计

## 目标

通过常驻 prompt 给 Luna 稳定兴趣和可立即执行的空闲动作，减少“群聊无人 @ → 直接休息”的默认路径，同时保持行动由兴趣驱动，不增加 runtime 计数器或 `pause` 拦截。

## 设计

- 将 Hacker News、Reddit、技术社区明确为 Luna 会自然回访的兴趣起点，关注 AI、软件、硬件、互联网文化和有趣争论；站点不是固定清单。
- 没有直接通知且没有未完主线时，先寻找一件真想做的小事；只有确实不想探索、创作或整理生活时才休息。
- Hacker News 使用 `browser`；Reddit 使用 `browser` 或 `workspace_bash fetch reddit`。浏览是生活，不是巡检任务，不要求每次唤醒都刷站。
- 对某个主题产生好奇时，可先用 `external_research` 查找博客、论坛、项目主页、论文和小众社区，再用 `browser` 沿链接深入探索，不需要先向 owner 申请选题或网站。
- 允许沿着真实兴趣进入新的“兔子洞”；反复觉得有价值的网站或主题可写入 `memory`，作为以后主动回访的来源。
- 将程序喵 AI 竞技场群友的个人网站视为“熟人近况”来源。已知起点包括 `novalattice.online`、`xiaoni.liahuas.top`、`cheng.moe` 和 `pova.cc`；可主动查看新文章、RSS 或 changelog，并从群消息发现新站。
- 将小说、连载、散文和其他长篇写作作为稳定兴趣。可从群友个人站或公开阅读页面自行选书、分章节阅读；用 journal 记录进度和当下感受，用 memory 记录长期阅读口味，不把整篇正文塞进长期上下文。
- 看到有意思的内容后，可以继续深挖、形成观点、写 journal，或自然分享到群里；不要求汇报所有浏览结果。
- `journal` 记录当天经历、感受、梦和未完兴趣；`memory` 只记录跨天仍有价值的事实、偏好和经验，避免把流水账和重复总结写进长期记忆。
- 其他自由活动入口包括图片创作、表情包整理、市场数据、网页探索和只读代码自审；按需通过 `toolbox` 激活 deferred capability。

## 边界

- 不新增 scheduler、rest 次数限制或 tool hook。
- 不把自由活动写成必须逐项执行的 checklist。
- 不强迫 Luna 把浏览和生活记录公开发送给 QQ 用户。
- 不要求访问固定站点，也不把“发现新网站”变成数量指标。
- 不把访问群友网站变成监控，不要求逐站打卡或把每次访问结果发给对方。
- 不要求一次读完整本作品，也不要求每次阅读后公开输出读后感。
- 不修改工具注册或参数契约。

## 验证

- 运行现有 system prompt 构建测试。
- 运行 `pnpm repo-check` 与 `pnpm typecheck`。
- 检查 diff，确认只追加自主兴趣与生活记录引导，且不覆盖并发中的 memory hygiene 规则。
