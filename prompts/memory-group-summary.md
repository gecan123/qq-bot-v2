请根据给出的历史 JSON 和记忆增量消息，更新群聊记忆，并严格返回符合 schema 的 JSON。

字段要求：
1. `summary`：2 到 3 句话总结这个群当前的整体印象，覆盖氛围、常见话题、组织方式或互动风格，要求客观、可复用、便于后续拼接到 prompt 中。
2. `topics`：列出 2 到 6 个近期稳定出现或明显重要的话题关键词，短语即可，不要写成长句。
3. `activePatterns`：列出 1 到 4 条成员活跃规律、时间特征、互动模式或组织方式观察。
4. `styleTags`：列出 2 到 6 个简短标签，例如 `热闹`、`务实`、`爱整活`、`夜间活跃`。

更新规则：
1. 保留旧 JSON 里仍然成立的观察，补充新增信息，删除已经明显过时的内容。
2. 只基于消息里能观察到的事实做归纳，不要脑补群定位、成员关系或长期背景。
3. 如果消息量有限，就保守更新，不要为了凑字段而编造。
4. 标签和话题尽量去重，避免同义反复。

输出要求：
1. 只输出符合 schema 的 JSON，不要输出 Markdown，不要补充解释。
2. 所有字段都必须返回；没有把握时返回更短、更保守的内容。
