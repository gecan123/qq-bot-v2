请根据历史 JSON 和记忆增量消息，更新这个人的用户画像，并严格返回符合 schema 的 JSON。

字段要求：
1. `profile`：2 句话以内概括这个人的稳定印象，覆盖性格倾向、兴趣关注点或常见角色，不要写成流水账。
2. `traits`：列出 2 到 5 个稳定特征关键词，例如 `直接`、`爱开玩笑`、`执行型`。
3. `interests`：列出 1 到 5 个从发言中能看出的兴趣、关注点或常参与话题。
4. `speakingStyle`：列出 2 到 5 个说话风格标签，例如 `短句确认`、`高频反问`、`口语化`。
5. `examples`：挑选 3 到 5 句最能代表其表达方式的原话，尽量保留原始措辞；如果本轮消息不够，可沿用仍然成立的旧例句。

更新规则：
1. 保留旧 JSON 中仍然成立的观察，补充新信息，修正过时判断。
2. 只根据消息本身做归纳；没有证据不要推断职业、年龄、关系或立场。
3. `examples` 必须是原话，不要改写成总结句。
4. 如果信息不足，就输出更保守、更泛化的画像，不要编造。

输出要求：
1. 只输出符合 schema 的 JSON，不要输出 Markdown，不要补充解释。
2. 所有字段都必须返回，数组允许较短但不要留空字符串。
